

# Enable Google OAuth Single Sign-On

## Overview

Add a "Sign in with Google" option to the login page, allowing users to authenticate with their Google accounts. This is fully managed by Lovable Cloud - no additional Google Cloud Console setup required.

---

## What Will Change

### Login Page Updates
The login page will get a new Google sign-in button above the existing email/password form:

```
┌─────────────────────────────────────┐
│   [G] Continue with Google          │
└─────────────────────────────────────┘

         ─────── or ───────

      [Email/Password Form]
```

### User Experience
- **New users**: Click Google button → Google consent screen → Redirected to dashboard → Profile auto-created
- **Existing users**: If their Google email matches an existing account, they can sign in seamlessly
- **Institutional accounts**: Works with Google Workspace (e.g., `@university.edu`)

---

## Technical Implementation

### Step 1: Configure Google Provider
Run the Lovable Cloud social auth configuration tool to:
- Enable Google OAuth in the authentication settings
- Generate the `@lovable.dev/cloud-auth-js` integration module
- Create the `src/integrations/lovable/` folder with auth utilities

### Step 2: Update Login Page
Modify `src/pages/LoginPage.tsx` to add:

```typescript
import { lovable } from "@/integrations/lovable/index";

const handleGoogleLogin = async () => {
  setIsLoading(true);
  const { error } = await lovable.auth.signInWithOAuth("google", {
    redirect_uri: window.location.origin,
  });
  if (error) {
    setError("Google sign-in failed. Please try again.");
    setIsLoading(false);
  }
};
```

Add a styled Google button with divider above the email form.

### Step 3: Handle New Google Users
The existing `AuthContext` already handles `onAuthStateChange`, so new Google-authenticated users will:
1. Trigger the auth state listener
2. Attempt to fetch their profile
3. If no profile exists, we'll need to handle profile creation

**Consideration**: Currently, profiles are created via admin invitation. For Google SSO users, we may need to either:
- Auto-create a minimal profile on first Google login
- Or require Google users to be pre-invited (email must exist in profiles table)

## Security Notes

- Uses Lovable Cloud's managed Google OAuth credentials (secure by default)
- No API keys need to be stored in the codebase
- Users can optionally configure their own Google OAuth credentials for custom branding via Cloud dashboard

---

## Post-Implementation

After enabling, you can customize Google OAuth settings (like restricting to specific domains) via the Lovable Cloud dashboard under **Users → Authentication Settings → Google**.
